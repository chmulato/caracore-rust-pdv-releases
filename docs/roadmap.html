Princípios do roadmap

Prioridade 1 Balcão: vender rápido, pagar de forma flexível, corrigir carrinho, pós-venda no mesmo fluxo.
Prioridade 2 Estoque e cadastro: produto, entrada, alerta de mínimo, histórico básico.
Longo prazo SEFAZ homolog real, multi-loja, inventário cíclico — só quando o negócio pedir.
Manter LGPD, auditoria, conciliação e arquitetura Rust — diferenciais já superiores ao legado.

FASE 1 P0 · Balcão Entrega: VendasPageV2

Fase 1 — Loja funciona no balcão

Unificar a experiência no V2 e portar o melhor do legado Java (NovaVendaController, FormaPagamentoDialog) sem reativar duas telas de venda.

#EntregaInspiração legadoNotas técnicas (Rust)
1.1 Carrinho editável — alterar qty, remover linha, subtotal ao vivo OK Carrinho persistente + spinner por linha CheckoutCarrinhoPanel + useVendasCheckout
1.2 Pagamento misto — dinheiro + cartão + PIX na mesma venda; troco OK FormaPagamentoDialog IPC registrar_pagamentos_venda_operacional + CheckoutPagamentoPanel
1.3 Desconto — item e venda (%), gate gerente/admin OK aplicarDescontoVenda + desconto por linha canApplyDesconto + distribuição proporcional no IPC
1.4 Scanner robusto — lookup por código no banco (não só lista carregada) OK BIPE + catálogo filtrável buscar_produto_por_codigo_pdv + fallback no hook
1.5 Pós-venda no V2 — cancelar, devolver, troca acoplada OK VendasConsultaTab + fluxos pós-venda CheckoutPosVendaPanel (dupla confirmação)
1.6 Cadastro rápido de produto — código, nome, preço, estoque inicial OK CadastroProdutoTab cadastrar_produto_rapido_pdv + Sheet no V2
1.7 Observação e última venda — campo na venda; banner pós-checkout OK Caixa “última venda” + observação Campo observacao + banner ultimaVenda no checkout

Critério de aceite — Fase 1

  • Operador conclui venda multi-item com pagamento misto e troco sem sair de /vendas-v2.
  • Gerente aplica desconto com perfil autorizado; auditoria registrada.
  • Scanner encontra produto por código mesmo fora da grade visível.
  • Cancelamento/devolução acessível no V2 (não em aba escondida).
  • Cadastro de produto novo na loja sem SQL manual.
  • Testes UI: vendas-v2-operacional.ui.test.tsx + regressão IPC.
FASE 2 P1 · Gestão PME Entrega: GestaoPage /gestao

Fase 2 — Dono enxerga o negócio

Retaguarda comercial enxuta — sem fiscal pesado. Inspirado em DashboardTab e RelatoriosTab do legado. Rota /gestao com abas Dashboard, Vendas, Relatórios e Kardex.

#EntregaValor para PME
2.1 Dashboard comercial — vendas hoje, ticket médio, top produtos, estoque baixo OK Visão diária na abertura da loja
2.2 Consulta de vendas — filtro por data, código, total; detalhe e reimpressão OK Atendimento pós-venda e conferência
2.3 Relatórios leves — ranking produtos, estoque abaixo do mínimo (CSV; PDF diferido) OK Substituto pragmático de ABC/ranking fiscal
2.4 Estoque mínimo — campo no produto + alerta no dashboard OK Single-loja primeiro (multi-loja depois)
2.5 Histórico de movimentações — kardex por produto (entrada, venda, ajuste) OK Tabela movimentacao_estoque já existe
2.6 EstoquePage melhorada — busca FTS no picker, não só dropdown OK Catálogos com centenas de SKUs
2.7 Cupom não fiscal — impressão/spool ligado ao checkout (MOCK; porta real diferida) OK Auto cupom pós-venda + reimpressão em /gestao

Critério de aceite — Fase 2

  • Gestor abre dashboard e identifica estoque crítico e vendas do dia sem exportar SQL.
  • Relatório ranking exportável em um clique (CSV).
  • Histórico de movimentação consultável por produto nos últimos 30 dias.
  • Testes UI: gestao-operacional.ui.test.tsx + vendas-v2-operacional.ui.test.tsx (7/7).
  • Polish UI responsivo (pós-Fase 2): ciclo ENCERRADO (sec. 105–106) — tablet + mobile + caixa turno/relatório + clientes + CaixaPage legado (/caixa, caixa-flow); testes caixa 15/15. Diferido: PDF, impressora real, accordion LGPD mobile.

Comparativo — legado Java × Rust v3

O que copiar (UX/regra de negócio) vs o que já não regredir (plataforma).

Portar do legado

  • Pagamento misto e troco
  • Descontos com perfil gerente
  • Carrinho editável e venda recuperável
  • Dashboard e ranking comercial
  • Cadastro produto/estoque integrado
  • Consulta de vendas + cupom

Manter no Rust (já superior)

  • LGPD checkout + gestão titular/colaborador
  • Auditoria e governança operacional
  • Conciliação CSV/OFX/JSON
  • Atalhos configuráveis (F1–F12)
  • Confirmação em duas etapas (estoque/caixa)
  • Backend estorno/devolução/troca maduro

Explicitamente fora do escopo imediato

  • Homologação SEFAZ como bloqueador de release — trilha Fase 3.
  • Duas telas de venda (/vendas + /vendas-v2) — V1 será absorvida pelo V2.
  • Multi-loja antes de single-loja redondo.
  • Relatórios fiscais complexos antes do dashboard comercial básico.
